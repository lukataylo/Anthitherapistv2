# Reframe — replit.md

## Overview

Reframe (internal name: Antitherapist) is a mobile-first CBT (Cognitive Behavioural Therapy) self-practice app. Users type or speak a negative thought; the app uses Claude AI (Anthropic) to analyse it word-by-word, tagging each word with a cognitive distortion category. Distorted words are shown as coloured tappable chips inline in the thought. Users then work through reframing those words via an interactive GamePanel or five standalone mini-games, all of which are personalised using the user's own captured history.

**Key features:**
- Multi-turn journal mode on home screen with U-shaped therapeutic conversation (descent → threshold → ascent)
- Synchronous pattern matching (18 patterns, 13 categories) with background Claude analysis (`/api/analyse-turn`)
- Bold typographic suggested questions below input, adapting to conversation depth
- Session feedback view with highlighted sentences (amber matched text), category labels, tap-to-expand reframe hints
- AI-powered word-level thought annotation (Claude Sonnet via `/api/reframe`)
- Four distortion categories: `belief`, `fear`, `absolute`, `self_judgment`
- Reframing game panel with free-text input, hints, 50/50, and skip
- Five CBT mini-games: Sort Tower, Rocket Reframe, Thought Check, Mind Voyage (SailGame), Reword
- Daily streak tracking (AsyncStorage)
- Socratic Discuss mode (chat with Claude)
- Insights dashboard with donut/trend charts and AI-generated pattern observations
- **Belief flashcard Practice tab** — sequential deck of cards built from reframing history, journal insights (via `/api/discuss/insights`), and open-ended belief prompts. Users flip cards, type answers for prompts, and use Previous/Next navigation to drill positive beliefs. Progress is tracked via a position indicator (e.g. 3/12).
- Dark glassmorphic visual design throughout

**Monorepo workspace structure:**

## System Architecture

### Mobile App (`artifacts/mobile`)

**Framework:** Expo SDK 54, React Native 0.81, Expo Router 6 (file-based routing).

**Navigation:** Four main tabs (Reframe `/`, History `/history`, Practice `/flashcards`, Discuss `/discuss`) managed by Expo Router. The Discuss tab is hidden from the tab bar (`href: null`). A custom `TabBar` component replaces the default tab bar with a floating glassmorphic BlurView pill. Tab routes are defined in `app/_layout.tsx`. Insights (charts, trends, AI patterns) are embedded in the History tab header via the `InsightsSection` component.

**State management:** React Context only — no Redux or Zustand:
- `JournalSessionContext` — multi-turn journal session state (active session, turns, analyses). Persisted to AsyncStorage (`temet_active_session`, `temet_active_turns`, `temet_active_analyses`). Completed sessions stored in `temet_session_history`. Manages arc controller (U-shaped descent/ascent), check-in states, and feedback assembly.
- `GameContext` — active reframing session (current thought, word analysis, which screen is visible, active word being reframed). Session-only, not persisted.
- `HistoryContext` — persistent log of up to 100 past entries in AsyncStorage (`reframe_history_v1`). Provides `addEntry`, `updateEntry`, `removeEntry`.
- `StreakContext` — daily streak counter in AsyncStorage (`reframe_streak_v1`). `recordReflection()` is called when a journal session ends.

Provider nesting order (outermost first): `SafeAreaProvider → ErrorBoundary → QueryClientProvider → GestureHandlerRootView → HistoryProvider → StreakProvider → GameProvider → JournalSessionProvider`.

**API communication:** Auto-generated React Query hooks from `@workspace/api-client-react` (generated by Orval from the OpenAPI spec). `setBaseUrl()` is called at module load in `_layout.tsx` using `EXPO_PUBLIC_DOMAIN` env var.

**Animation:** React Native Reanimated 4 for complex animations (game mechanics, LetterTumble celebration, ThinkingAnimation orbital dots). Plain `Animated` API for simpler tab bar animations. `expo-haptics` for tactile feedback.

**Gesture handling:** `react-native-gesture-handler` with Reanimated worklet-thread gesture handling (used in SortTowerGame card swipes).

**Fonts:** Inter (400/500/600/700) loaded via `@expo-google-fonts/inter`. Splash screen held until fonts ready.

**Speech input:** `expo-speech-recognition` for voice capture. Gracefully hidden on web.

**Storage:** `@react-native-async-storage/async-storage` for all client-side persistence (history, streak). No server-side user accounts.

**Home screen state machine (CaptureScreen):**
```
mood-select (emoji picker) → journaling (multi-turn)
  → (tap "Wrap up" after 2+ turns) → wrapping (3 reflection questions)
  → loading-summary (Claude processes) → summary (insight cards with agree/disagree)
  → (start fresh) → mood-select
```
Features: emoji mood selector (happy/okay/sad/stressed/angry), animated question with "t" avatar above input, progressive opacity fade on older turns, emotional word scoring for check-ins (CHECKIN_THRESHOLD=12), LLM session summary via `/api/summarise-session`.

**Reframe state machine (GameContext — used by history/review flow):**
```
capture → (API returns) → cloud → (tap distorted word) → game → (reframe/skip) → cloud
```

**Mini-game architecture:** All five games are full-screen modal components launched from the History tab's `GameCarousel`. Each game:
- Calls a `buildDeck`/`buildRounds`/`buildQuestions` function that extracts content from `HistoryEntry[]` (personalisation)
- Falls back to curated content when history is sparse
- Implements its own score/combo system
- Renders an end-of-game summary inline (no shared summary component)

**Production build:** `scripts/build.js` triggers Metro bundler to produce a static web build in `static-build/`. `server/serve.js` (Node built-ins only) serves static files and proxies `/api/*` to the API server. `server/start-production.js` orchestrates both processes with `spawn`.

### API Server (`artifacts/api-server`)

**Framework:** Express 5.

**Structure:** Single `app.ts` factory with ordered middleware (pino-http → cors → rate-limit → express.json → `/api` router). Entry point `index.ts` reads `PORT` env var and starts the server.

**Routes:** All under `/api` prefix. Includes `/api/healthz` health check, `/api/reframe` (Claude thought analysis), `/api/analyse-turn` (background therapeutic analysis for journal turns), `/api/patterns` (AI pattern observations for insights screen), `/api/discuss` (Socratic chat), `/api/summarise-session` (Claude session summary + insight cards for journal wrap-up), `/api/discuss/insights` (AI-extracted flashcard insights from recent discuss sessions), `/api/conversations` and `/api/conversations/:id/messages`, and a landing page router.

**Logging:** Pino with `pino-pretty` in development, raw JSON lines in production. Sensitive headers (`Authorization`, `Cookie`, `Set-Cookie`) are redacted. Query strings stripped from logged URLs to avoid logging user thoughts.

**Build:** esbuild bundles the server for production. An allowlist of npm packages are bundled to reduce cold start syscalls; remaining deps are left external.

### Database (`lib/db`)

**ORM:** Drizzle ORM with PostgreSQL dialect. `drizzle-kit` for schema push (`pnpm run push`).

**Connection:** `pg` Pool, URL from `DATABASE_URL` env var. The `db` object and all schema exports are re-exported from `lib/db/src/index.ts`.

**Schema:** Defined in `lib/db/src/schema/index.ts` (not shown in full, but referenced by API server and drizzle config).

**Note:** Most user data (thoughts, history, streak) is stored client-side in AsyncStorage. The database is used for server-side concerns (likely session/user data if auth is added, or server-persisted patterns).

### API Spec & Code Generation (`lib/api-spec`, `lib/api-client-react`, `lib/api-zod`)

**Spec:** OpenAPI YAML at `lib/api-spec/openapi.yaml`.

**Codegen:** Orval generates two outputs from the spec:
1. `lib/api-client-react/src/generated/` — React Query hooks with a custom fetch client (`custom-fetch.ts`) that supports `setBaseUrl()` and `setAuthTokenGetter()`.
2. `lib/api-zod/src/generated/` — Zod schemas for server-side request validation.

**Pattern:** API contracts are defined once (OpenAPI) and consumed as type-safe generated code on both client and server. Never hand-write API types.

### Mockup Sandbox (`artifacts/mockup-sandbox`)

**Purpose:** Isolated Vite + React + Tailwind + shadcn/ui sandbox for UI prototyping. Unrelated to the production mobile app.

**Auto-discovery:** `mockupPreviewPlugin.ts` (a Vite plugin) watches `src/components/mockups/**/*.tsx` and generates a module manifest for live preview routing.

### Anthropic AI Integration (`lib/integrations/anthropic-ai`)

Thin wrapper around `@anthropic-ai/sdk` with retry (`p-retry`) and concurrency limiting (`p-limit`). Used only by the API server — never imported by the mobile app directly.

## External Dependencies

### Anthropic Claude API
- **Purpose:** Word-level cognitive distortion classification (`/api/reframe`), reframe suggestions, Socratic coaching (`/api/discuss`), pattern observations (`/api/patterns`).
- **SDK:** `@anthropic-ai/sdk` via `@workspace/integrations-anthropic-ai`.
- **Auth:** `ANTHROPIC_API_KEY` environment variable (server-side only).

### PostgreSQL Database
- **Purpose:** Server-side persistent storage (schema managed via Drizzle).
- **Connection:** `DATABASE_URL` environment variable.
- **Client:** `pg` (node-postgres) Pool.

### AsyncStorage (Client-side)
- **Purpose:** All user-facing persistence — thought history (up to 100 entries), streak data.
- **Keys:** `reframe_history_v1`, `reframe_streak_v1`.
- **Library:** `@react-native-async-storage/async-storage`.

### Expo Platform Services
- `expo-blur` — BlurView for glassmorphic tab bar and modals (iOS/Android only; CSS fallback on web).
- `expo-haptics` — Haptic feedback on game interactions.
- `expo-speech-recognition` — Voice input for thought capture.
- `expo-font` — Font loading (Inter family).
- `expo-router` — File-based navigation.
- `expo-linear-gradient` — Gradient backgrounds in games.
- `expo-splash-screen` — Holds splash until fonts load.

### React Query (`@tanstack/react-query`)
- **Purpose:** API mutation and caching layer in the mobile app. Used via auto-generated hooks from `@workspace/api-client-react`.

### React Native Gesture Handler + Reanimated
- `react-native-gesture-handler` — Pan gesture for SortTowerGame card swipes.
- `react-native-reanimated` — Complex animations (orbital loading, letter tumble, game mechanics). Runs on a separate worklet thread.

### Orval
- **Purpose:** OpenAPI → React Query hooks + Zod schemas code generation.
- **Run:** `pnpm run codegen` from `lib/api-spec`.

### Build / Dev Tooling
- `pnpm` v10 — workspace package manager (enforced).
- `TypeScript` ~5.9 — strict mode throughout.
- `esbuild` — API server production bundling.
- `tsx` — TypeScript execution for dev server and scripts.
- `Vite` — Mockup sandbox dev server and build.
- `pino` / `pino-http` — Structured logging on API server.
- `drizzle-kit` — Database schema management.
